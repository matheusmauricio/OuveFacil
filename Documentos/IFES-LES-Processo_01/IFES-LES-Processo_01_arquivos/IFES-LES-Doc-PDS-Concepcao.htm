Documentação da Fase de Concepção

 

 

 

 

 

A fase de concepção consiste em uma etapa na qual o analista vai buscar as primeiras informações sobre o sistema a ser desenvolvimento. Nessa etapa, assume-se pouco conhecimento do analista sobre o sistema e uma grande interação com o usuário e cliente.

 

01 Atividade: Levantar Requisitos (Visão Geral)

Finalidade

  • Identificar as necessidades do cliente quanto ao sistema a ser desenvolvido, documentando os requisitos no Documento de Requisitos, o qual representa o contrato de comprometimento com o cliente.

Artefatos de Entrada:

  • Template do Documento de Requisitos

Artefatos de Saída:

  • Documento de Requisitos

Papel: Analista de Sistema

Ferramentas: Word ou Writer

Passos:

1.      Preencher a Seção 1 Introdução do Template do Documento de Requisitos;

1.1.   Esta seção tem a finalidade de explicar sobre o documento em questão. Portanto, está praticamente preenchida, bastando inserir o nome do Sistema;

2.      Preencher a Seção 2 Visão Geral do Template do Documento de Requisitos;

2.1.   Nesta seção deve-se fornecer, de forma geral, o contexto principal de funcionamento do sistema;

3.      Armazenar o Documento de Requisitos;

3.1.   Salvar com o nome <Sigla do Projeto>-DocumentoDeRequisitos_01

 

 

02 Atividade: Elaborar Modelo Conceitual

Finalidade

  • Construir o Modelo Conceitual. Neste modelo serão identificadas as principais classes do sistema, bem como seus relacionamentos. Nesta etapa não será necessário a identificação dos atributos das classes.

Artefatos de Entrada:

  • Documento de Requisitos

Artefatos de Saída:

  • Modelo Conceitual

Papel: Analista ou Programador

Ferramentas: Word ou Writer; Astah

Passos:

  1. Ler Documento de Requisitos;
  2. Criar os pacotes de organização do projeto;

a)      view (referente a interface gráfica do sistema)

b)      control (referente as classes de controle do sistema)

c)      dao (referente as classes que proveem acesso ao banco)

d)     model (referente as classes que representam entidades do sistema, ou seja, os chamados beans em java. Classes que possuem apenas atributos privados, e métodos set e get para acessarem os mesmos).

03 Atividade: Levantar Requisitos

Finalidade

  • Identificar as necessidades do cliente quanto ao sistema a ser desenvolvido, documentando os requisitos no Documento de Requisitos, o qual representa o contrato de comprometimento com o cliente.

Artefatos de Entrada:

  • Documento de Requisitos (Versão 01)
  • Modelo Conceitual

Artefatos de Saída:

  • Documento de Requisitos (Versão 02)

Papel: Analista de Sistema

Ferramentas: Word ou Writer

Passos:

1.      Preencher a Seção 3 Requisitos Funcionais do Template do Documento de Requisitos;     

1.1.   Esta é a principal seção deste documento. Para seu preenchimento sugere-se, primeiramente, que o analista identifique as possíveis classes, considerando as quatro categorias seguintes:

1.1.1.      Atores (Usuários do Sistema);

1.1.2.      Coisas (Fitas, Filmes, etc.);

1.1.3.      Lugares (Bairro, Cidade, etc.);

1.1.4.      Eventos (Processos de Negócio: Empréstimo, Reserva);

2.      Preencher a Seção 4 Requisitos Não Funcionais do Template do Documento de Requisitos;

Observações: para que todos os grupos de alunos da disciplina de Laboratório de Engenharia de Software tenham projetos com complexidades similares vamos estabelecer números mínimos de requisitos funcionais para as três categorias:

·         Manutenção de Cadastros (CRUD): no mínimo 7 conceitos;

·         Processos e Negócio: no mínimo 4 conceitos;

·         Relatórios: no mínimo 3 requisitos de listagens com totalizações ou filtros (exemplo: Listar Empréstimos por Cliente e Período; Listar Total de Empréstimos por Cliente e Período, etc.);

3.      Armazenar o Documento de Requisitos;

3.1.   Salvar com o nome <Sigla do Projeto>-DocumentoDeRequisitos_02

 

 

 

 

04 Atividade: Organizar Requisitos

Finalidade

  • Organizar todos os requisitos identificados e documentados na atividade anterior. Estes requisitos devem ser organizados em: Processos de Negócios, Manutenção de Cadastros (CRUD) e Relatórios/Listagens.

Artefatos de Entrada:

  • Documento de Requisitos
  • Template do Documento de Organização de Requisitos

Artefatos de Saída:

  • Documento de Organização de Requisitos

Papel: Analista de Sistema

Ferramentas: Excel ou Calc

Passos:

  1. Identificar os requisitos detalhados no Documento de Requisitos;
  2. Preencher o Template do Documento de Organização de Requisitos;
  3. Armazenar Documento de Organização de Requisitos;

a)      Salvar com o nome <Sigla do Projeto>- DocumentoDeOrganizaçãoDeRequisitos_01
